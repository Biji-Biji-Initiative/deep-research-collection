# Examples

This folder contains example structures and templates to demonstrate how to organize deep research reports.

## Purpose

- Demonstrate proper folder structure
- Show example content format
- Provide reference for contributors
- Test templates and workflows

## Structure

```
examples/
└── mereka-lms/
    ├── README.md                    # Repository-specific index
    └── deep-research/
        └── sample-research/
            ├── prompt.md            # Research question
            └── result.md            # Research findings
```

## How to Use

1. **As Reference**: Copy the structure when adding new repositories
2. **As Template**: Use the sample files as starting points
3. **As Testing**: Practice the contribution workflow here first

## Note

The `examples/` folder is for demonstration only. Actual research should be added to repository-specific folders at the root level, not in `examples/`.

## Cleaning Up

Once you're familiar with the structure, you can:
1. Keep examples for future reference
2. Remove them to keep the repository clean
3. Move them to a separate documentation repository
